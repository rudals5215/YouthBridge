## 1. 왜 만들었나요?

청년 지원 정책은 정부, 지자체, 공공기관 등 여러 곳에 흩어져 있어요. 어떤 혜택을 받을 수 있는지 알려면 일일이 검색해야 하고, 자신에게 해당되는 정책인지 판단하기도 어려워요.

**YouthBridge**는 이 문제를 해결하기 위한 서비스예요. 공공데이터포털 API에서 청년 정책 데이터를 수집해서 지역, 나이, 관심 분야로 필터링할 수 있게 만들어요.

---

## 2. 핵심 기능

```
서비스 사용자
├── 정책 목록 조회   — 지역, 나이, 카테고리, 키워드로 필터링
├── 정책 상세 조회   — 지원 대상, 지원 내용, 신청 방법, 마감일
├── 맞춤 추천        — 질문에 답하면 나에게 맞는 정책 추천
├── 즐겨찾기         — 관심 정책 저장 (마이페이지에서 관리)
└── 회원가입/로그인  — 이메일 + 카카오 소셜 로그인

관리자
├── 대시보드         — 회원 수, 정책 수 등 통계
├── 회원 관리        — 회원 목록 조회
├── 정책 관리        — 정책 수동 수정/삭제
├── 공공 API 동기화  — 버튼 한 번으로 최신 데이터 수집
└── 공지사항 작성
```

---

## 3. 기술 스택

| 분류 | 기술 | 선택 이유 |
|---|---|---|
| 프론트엔드 | React + TypeScript | 컴포넌트 재사용, 타입 안정성 |
| 빌드 도구 | Vite | CRA보다 빠른 개발 서버 |
| 라우터 | React Router | SPA 표준 라우팅 |
| 상태 관리 | Zustand | Redux보다 간단한 전역 상태 |
| HTTP 클라이언트 | axios | 인터셉터로 토큰 자동 처리 |
| 백엔드 | Spring Boot 3 (Java) | 취업 시장에서 가장 많이 쓰는 스택 |
| DB | MariaDB | MySQL 호환, 오픈소스 |
| ORM | Spring Data JPA | SQL 없이 메서드 이름으로 쿼리 생성 |
| 인증 | JWT | 서버 세션 없이 stateless 인증 |
| 아이콘 | Remixicon | 무료, 디자인 퀄리티 높음 |
| 폰트 | Noto Sans KR | 한글 가독성 최적화 |

---

## 4. 개발 순서

프론트엔드를 먼저 완성한 뒤 백엔드를 연결하는 방식으로 진행해요.

```
1단계  프론트엔드 UI 완성  (더미 데이터로 동작 확인)
2단계  백엔드 API 구현     (DB 설계 → 인증 → 정책 → 즐겨찾기)
3단계  프론트-백엔드 연결  (더미 데이터 → 실제 API 교체)
4단계  공공 API 크롤링     (실제 정책 데이터 수집)
5단계  배포               (Vercel + AWS EC2 + RDS)
```
